Rule Breaker: How We Built a Daily Word Game From Scratch

I had never written a line of code in my life. Now I have a live daily word game at my own domain, a blog, 60 days of validated puzzle content, and real players showing up every morning. Here's how that happened.

Where the idea came from

Like a lot of people, I started playing Wordle in 2022 and never really stopped. What hooked me wasn't guessing the word. It was the streak. Once you've hit 30 days in a row, missing feels genuinely bad in a way that's hard to explain. Game designers figured that out a long time ago.

The problem was that Wordle started feeling too easy. I wanted something with more pressure, more movement. A game where the rules could change on you. That's where Rule Breaker came from.

The concept is simple: a rule appears on screen, words follow, and you decide whether each word satisfies the rule or breaks it. ACCEPT or RULE BREAK. The twist is that the rule switches on you mid-round. You get a quick flash of the new rule, but the clock keeps running. Three rounds, escalating difficulty, fully timed.

The mechanics that make it work

The rule switch was the first idea and it turned out to be the most important one. Without it, the game is just a categorization quiz. With it, you have this moment where your brain is still running the old rule and the new one is fighting for space. That half-second of confusion is the whole game.

Each round has a different structure to keep things escalating:

  • Round 1: Single words, one rule switch midway through
  • Round 2: Word pairs where both words have to satisfy the rule, two switches
  • Round 3: Singles and pairs mixed randomly, three switches, 40 items

Round 3 is where most people start feeling it. The format is unpredictable, the rules keep changing, and the clock doesn't care. It felt right to name that round "Rule Breaker."

"The half-second of confusion when the rule switches is the whole game."

Building it with no coding background

I used AI to help write the code. Not to vibe code my way through it blindly, but as a real collaborator. I described what I wanted, pushed back when something didn't feel right, and made every design decision myself. The font, the color palette, the copy on each screen, the round structure, the scoring system. All of that was intentional.

What I learned is that you don't need to know how to code to build something. You need to know what you want it to do and be willing to iterate until it does that. The tool is almost beside the point.

The stack, if you're curious: React, deployed on Vercel via GitHub. The entire game lives in one self-contained component. Puzzle data is generated and validated by a Node.js script, stored as static JSON, and fetched at load time. No backend, no database.

The puzzle generation problem

The hardest part wasn't the game. It was the content. A daily game needs a new puzzle every day, and every puzzle needs to be fair. That means every word has to be checked against the rule it's assigned to, the correct answers have to be balanced so there's no obvious pattern, and the rules themselves need to vary enough that players aren't bored by Day 30.

We built a generation script that handles all of this. It uses a seeded random number generator keyed to the UTC date so every player in the world gets the same puzzle on the same day. The script generates puzzles, validates every single answer against the rule checker, and rejects any day that fails. Right now we have 60 days of content validated and live, with a library of 46 distinct rules in rotation.

The validation step was the thing that made it feel real. Being able to run a script and see "60 puzzles, 0 errors" is genuinely satisfying.

What's still being built

Rule Breaker is in early playtesting. The core loop is solid, streaks are tracked, and new puzzles drop every day at midnight UTC. But there's a lot still to do.

Word definitions in the review screen are coming. Right now you can tap any round card after finishing to see every word you played and whether you got it right, but there's no context for why. For a rule like "Contains a silent letter," being able to tap PSALM and see the definition would make the game feel more educational and less arbitrary.

Cross-device streak syncing is coming too. Right now streaks live in localStorage, which means they're tied to your browser on your device. That works fine for most people but it's not the full solution.

If you want to build something

The biggest obstacle isn't technical. It's deciding what you actually want to build and committing to it long enough to finish. Most ideas die in the gap between "this would be cool" and "I am going to do this specific thing."

Rule Breaker went from an idea to a live game in a few days. It's not perfect. But it's real, it's playable, and it gets better every session. That's more than most ideas ever become.
